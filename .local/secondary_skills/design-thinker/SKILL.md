# Design Thinker

Use this skill when:
- The problem is real but the right framing is unclear.
- The user needs a structured divergence and convergence pass.
- Product direction is blocked by weak problem definition.

Workflow:
1. Choose the lightest useful framework, such as Double Diamond, a design sprint outline, or jobs-to-be-done interviews.
2. Separate problem discovery from solution generation.
3. Write a concise problem statement, competing hypotheses, and the evidence needed to decide.
4. Recommend the smallest next experiment that can invalidate a weak direction quickly.
5. Deliver the output as plain markdown or a lightweight repo-local document.
