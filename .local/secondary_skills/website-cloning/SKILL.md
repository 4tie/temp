# Website Cloning

Use this skill when:
- The user wants a close recreation of a live website.
- The site must be studied through the browser, not just described from memory.
- The deliverable should be a local codebase, not a screenshot.

Rules:
1. Confirm the use is legitimate before cloning a site in detail.
2. Capture layout, copy, assets, breakpoints, and interactions before rebuilding.
3. Keep downloaded assets local so the clone does not depend on fragile remote URLs.
4. Use browser automation if the host supports it; otherwise inspect manually and rebuild progressively.
5. Validate both desktop and mobile behavior before calling the clone complete.
