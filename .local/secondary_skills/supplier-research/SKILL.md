# Supplier Research

Use this skill when:
- The user explicitly asks for help in this domain.
- The task centers on supplier research.
- The deliverable should be concrete rather than open-ended brainstorming.

Workflow:
1. Confirm the audience, goal, constraints, and exact output needed.
2. Gather repo context or current external facts as needed.
3. Produce a practical first draft that the user can review or reuse immediately.
4. If the host supports sub-agents, split independent research or drafting tracks; otherwise work sequentially.
5. Keep assumptions explicit so the user can approve, edit, or reuse the work easily.

Notes:
- Use current company and location data when availability or capacity matters.
