# Design Exploration

Use this skill when:
- The user wants options or variants.
- The best UI approach is not obvious yet.
- A page needs a brief before implementation starts.

Workflow:
1. Read the current page or component and list the constraints that must not change.
2. Write two to four meaningfully different directions in plain language.
3. Name the tradeoff each direction optimizes for.
4. If the host supports sub-agents, parallelize independent directions.
5. Translate the selected direction into concrete changes in `templates/` and `static/`.
